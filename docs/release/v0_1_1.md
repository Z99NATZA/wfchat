# v0.1.1 - Risk Review Hardening

Release date: 2026-07-04

Security, reliability, and compatibility hardening release.

## Highlights

- Hardened browser session auth to prefer the HTTP-only `wfchat_session` cookie.
- Protected admin endpoints with role-based session authorization.
- Added rate limits to cost-sensitive chat, speech, transcription, and image upload endpoints.
- Made chat message persistence transactional.
- Improved backend database error visibility.
- Fixed VOICEVOX speech-text translation compatibility with latest OpenAI text models.

## Security

### Admin Authorization

- Protected `/api/admin/*` routes with session and role authorization.
- Required `UserKind::Admin` for admin AI profile and provider status endpoints.
- Added backend route coverage for missing, guest, registered, and admin sessions.

### Cost-Sensitive API Rate Limits

- Added in-memory fixed-window rate limiting for expensive API endpoint
  families.
- Chat regular send and SSE streaming share one chat-message bucket.
- Assistant speech generation, user speech transcription, and image attachment
  upload use separate stricter buckets.
- Rate-limit keys prefer the `wfchat_session` cookie, then fall back to
  `X-WFChat-Session`; requests without a valid session identifier fall back to
  `X-Forwarded-For`, `X-Real-IP`, then an unknown-client bucket.
- Exceeded buckets return `429 Too Many Requests` with the normal JSON error
  body.

### Cookie-First Session Auth

- Added a shared backend session resolver that reads the HTTP-only
  `wfchat_session` cookie before `X-WFChat-Session`.
- Kept `X-WFChat-Session` as a compatibility fallback for non-browser or legacy
  local callers.
- Refreshed the session cookie from `GET /api/auth/me`, guest creation, Google
  login, and logout.
- Added `Secure` to session cookies when configured frontend origins include
  HTTPS, while keeping local HTTP development usable.
- Removed normal frontend reliance on browser-readable stored session ids and
  session headers for auth, chat, memory, attachments, speech, and sync calls.
- Added a frontend cookie-session bootstrap marker in `sessionStorage` and
  removed legacy `wfchat.sessionId` from localStorage when cookie auth is ready.

## Reliability

### Transactional Chat Persistence

- Wrapped chat message append, pending attachment linking, and chat timestamp update in one PostgreSQL transaction.
- Stopped ignoring write errors in the chat append path.
- Kept pending attachments unlinked and rolled back inserted messages when requested attachment linking is incomplete.
- Added backend regression coverage for the rollback behavior.

### Database Error Visibility

- Changed backend store methods to return explicit PostgreSQL `Result` values for session, auth/profile, chat, attachment, memory, and sync operations.
- Preserved expected not-found behavior as `404 Not Found` while letting real database failures become `500 database error`.
- Added database error logging through the API error boundary.
- Logged stale pending attachment cleanup failures instead of treating cleanup query failure as an empty cleanup batch.
- Updated backend route and integration tests for the new store error contract.

## Compatibility

### Voice Text Model Compatibility

- Shared the OpenAI text-model temperature compatibility rule between normal chat requests and VOICEVOX `japanese_translation` speech-text generation.
- Kept `temperature: 0.2` for translation models that support it.
- Omitted `temperature` for `gpt-5.5` and `gpt-5.5-*` translation requests.
- Kept text, TTS, and transcription model configuration documented as separate endpoint capabilities.
- Added backend regression coverage for the voice translation request payload.

## Documentation

- Updated backend, frontend, sync, SSE, image attachment, memory, voice, and
  Docker docs to match current runtime behavior.
- Documented cookie-first session ownership and rate-limit behavior.
- Documented transactional chat persistence and explicit backend database error
  propagation.
- Removed stale `DATA_PATH` configuration from `apps/api/.env.example`.
