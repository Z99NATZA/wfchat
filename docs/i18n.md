# Frontend i18n (UI Only)

## Scope
- This project uses frontend-only i18n for **UI text**.
- It does **not** auto-translate chat messages/content from users or AI.
- Devs fully control translations via locale files.

## Files
- `apps/web/src/i18n/index.tsx`: provider, locale state, `t()` helper, localStorage persistence.
- `apps/web/src/i18n/locales/en.json`: English UI strings (fallback locale).
- `apps/web/src/i18n/locales/th.json`: Thai UI strings.

## Rules
- Add new UI text with translation keys, not hardcoded strings in components.
- Keep key naming grouped by feature (example: `chat.header.*`, `chat.messageList.*`).
- If a key is missing in the active locale, `en` fallback is used.

## Usage
```tsx
import { useI18n } from "@/i18n";

function Example() {
  const { t } = useI18n();
  return <span>{t("chat.header.language")}</span>;
}
```

Interpolation:
```tsx
t("chat.messageList.emptyTitle", { name: "Aiko" })
```

## Language switch
- Locale is toggled from the header language button.
- Supported locales now: `en`, `th`.
- Selected locale is saved in localStorage key: `wfchat.locale`.
